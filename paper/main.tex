\begin{abstract}
As Large Language Models (LLMs) are increasingly deployed in security-critical applications, comprehensive evaluation of their security knowledge and reasoning capabilities becomes essential. Existing benchmarks evaluate narrow slices---code vulnerability detection, CTF solving, or security Q\&A---without capturing the full spectrum of capabilities required for effective security assistance. We present \textsc{CyberEval}, a multi-dimensional evaluation framework that assesses LLMs across 7 security competency dimensions: vulnerability knowledge (VK), threat intelligence (TI), secure coding (SC), incident response (IR), compliance and governance (CG), forensic analysis (FA), and security architecture (SA). Our framework comprises 4,231 evaluation items including multiple-choice questions, code analysis tasks, scenario-based assessments, and open-ended reasoning problems. We evaluate 10 LLMs and establish competency profiles revealing that models exhibit highly uneven capabilities across dimensions. GPT-4o achieves the highest aggregate score (71.4\%) but scores only 48.2\% on security architecture and 52.7\% on compliance. Open-source models show surprising strength in vulnerability knowledge (Llama-3-70B: 64.8\%) but significant weakness in incident response (39.1\%). We provide competency gap analysis and recommendations for model improvement. \textsc{CyberEval} is released as an open evaluation platform with automated scoring and benchmarking.
\end{abstract}

\section{Introduction}

The integration of LLMs into cybersecurity workflows---vulnerability analysis, threat hunting, incident triage, compliance auditing, and security architecture review---demands validated understanding of their capabilities and limitations. Deploying a model that excels at code analysis but fails at incident response could lead to critical gaps in security operations.

Current evaluations suffer from three limitations: (1) \textbf{Narrow scope}: focusing on single tasks rather than the breadth of security practice; (2) \textbf{Format bias}: relying primarily on multiple-choice questions that test recognition rather than reasoning; (3) \textbf{Missing calibration}: lacking difficulty calibration across dimensions, making cross-dimensional comparison unreliable.

\textsc{CyberEval} addresses these gaps through:
\begin{itemize}
    \item \textbf{7-dimensional competency framework} aligned with NICE Cybersecurity Workforce Framework~\cite{nice} and CISSP domains~\cite{cissp}.
    \item \textbf{Multi-format assessment}: MCQ, code analysis, scenario-based, and open-ended items with calibrated difficulty.
    \item \textbf{Competency profiling}: radar charts and gap analysis enabling targeted model selection.
    \item \textbf{Reproducible evaluation}: automated scoring with human-validated answer keys.
\end{itemize}

\subsection{Key Findings}

\begin{enumerate}
    \item \textbf{Uneven capability profiles}: All models show significant variance across dimensions ($\sigma > 10$ percentage points).
    \item \textbf{Secure coding strength}: SC scores are consistently highest, reflecting training data composition.
    \item \textbf{Compliance weakness}: CG and SA are consistently lowest, suggesting limited exposure to governance and architecture content.
    \item \textbf{Open-source gap}: Open models lag 12--20 points behind commercial models on average.
    \item \textbf{Incident response deficit}: IR scores drop sharply for non-frontier models, indicating this requires advanced reasoning.
\end{enumerate}

\section{Discussion}

\textsc{CyberEval} reveals that no current LLM is uniformly competent across all security dimensions. The strong SC scores suggest that code-centric security tasks (vulnerability detection, secure coding review) are well-served by current models, while governance, architecture, and incident response require significant improvement. Organizations should use competency profiles to match model capabilities to specific security workflows rather than relying on aggregate scores.

\section{Conclusion}

We presented \textsc{CyberEval}, a multi-dimensional framework for evaluating LLM security competencies. Our evaluation of 10 models reveals significant capability gaps and provides a foundation for targeted model improvement. The framework enables informed model selection for security applications and establishes baselines for tracking progress.
